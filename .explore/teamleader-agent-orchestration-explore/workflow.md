# 任务工作流

## 任务目标
- 探索如何构建一个用于多终端/多子代理协作的 Team Leader Agent。
- 关注长任务上下文续接、子终端错误监控、任务分派、冲突控制、交接记录和可视化观察面。
- 基于现有生态评估是直接使用现成工具、二次开发、还是自研更合适。

## 范围边界
- 范围内：
  - 调研现有相关方案与开源项目
  - 明确问题拆分与系统边界
  - 总结可行技术路线、优缺点、适用场景
  - 给出下一步选型或原型路径
- 范围外：
  - 本轮不实现 Team Leader Agent
  - 本轮不产出正式 proposal/design/tasks
  - 本轮不接入具体 ACP 服务端或 Zed 插件

## 成功标准
- 明确 Team Leader Agent 的核心职责和非职责。
- 形成一版可执行的技术路径比较。
- 识别关键难点：并行冲突、状态持久化、长上下文恢复、人工接管。
- 给出 2-3 条用户可选路线，便于后续进入提案或原型实现。

## 阶段规划
1. 识别目标问题与系统边界
2. 调研现有工具、协议与开源项目
3. 提炼架构层次和关键技术点
4. 形成路线选择建议与下一步

## 当前阶段
- 路由一：纯探索 / 需求梳理 / 方案比较

## 退出条件
- 已有一份足够供后续恢复的探索结论
- 已能决定下一步是继续提案还是直接做 MVP 原型
